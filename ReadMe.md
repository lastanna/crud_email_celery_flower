# Тестовое задание на доложность Python разработчика 
*Модель:* Создайте Django-модель Lesson (Урок) c полями на ваше усмотрение.

*Бизнес-логика:* Реализуйте механизм, который отслеживает важное событие в жизненном цикле урока — например, его создание или завершение.

*Фоновая задача (Celery):* Когда это событие происходит, должна асинхронно запускаться задача Celery, что будет имитировать отправку уведомления студенту (например, просто логировать сообщение: "Уведомление отправлено студенту [ID студента] по уроку [Название урока]").

*Что предоставить:* Ссылку на Git-репозиторий (GitHub/GitLab) с вашим решением. Краткий README.md с инструкциями по запуску (включая Celery) обязателен.

## Инструкция по запуску
1. Клонировать проект через git clone https://github.com/lastanna/crud_email_celery_flower.git

2. Первый раз запустить через
```commandline
docker compose up --build
```
Последующие запуски через
```commandline
docker compose up
```
3. Войти в систему через админ панель: http://127.0.0.1:8000/admin/
```
Имя пользователя: admin
Пароль: adminpass
```
4. Перейти на: http://127.0.0.1:8000/
5. Добавляем, редактируем уроки. Уведомления отправляются email-ами сервис использует тестовый файловый почтовый бэкенд. Письма сохраняются в корне проекта,
в папке tmp/app-messages, .log файлы в формате eml. Их можно прочитать просмотрщиками почты Outlook, 
Thunderbird и т.п. или онлайн сервисом, например, https://www.emlreader.com/
6. В задание также включила дашборд для мониторинга celery - Flower.
Отслеживать задачи можно тут:
http://127.0.0.1:5555/
